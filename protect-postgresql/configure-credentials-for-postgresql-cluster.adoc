= PostgreSQL 클러스터에 대한 자격 증명 구성
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
데이터 보호 작업을 수행하려면 PostgreSQL 클러스터에 대한 클러스터 사용자 자격 증명을 구성해야 합니다.

.시작하기 전에
* PostgreSQL 클러스터에 유효한 자격 증명이 구성되어 있지 않으면 자동 검색된 리소스에만 빨간색 자물쇠 아이콘이 표시됩니다. 이후 검색 작업 중에 구성된 자격 증명이 올바르지 않거나 클러스터 자체에 대한 액세스 권한을 제공하지 않는 것으로 확인되면 빨간색 자물쇠 아이콘이 다시 나타납니다.
* 리소스를 보호하거나 리소스 그룹에 추가하여 데이터 보호 작업을 수행하려면 유효한 PostgreSQL 클러스터 자격 증명을 구성해야 합니다.
* 자격 증명에 구성된 PostgreSQL 사용자는 클러스터에 액세스하고 백업 및 복원 작업을 수행할 수 있는 충분한 권한을 가지고 있어야 합니다. 최소한 해당 사용자는 *SUPERUSER* 역할 또는 그에 상응하는 권한(예: *pg_read_all_data*, *pg_write_all_data* 및 *replication* 권한)을 보유해야 합니다.
* SnapCenter는 구성된 자격 증명을 사용하여 *psql* 또는 *libpq* 클라이언트를 통해 PostgreSQL 인스턴스에 연결합니다. 플러그인 호스트에 PostgreSQL 클라이언트 도구가 설치되어 있는지 확인하십시오.



NOTE: 클러스터를 구성하기 전에 먼저 PostgreSQL 자격 증명을 SnapCenter에 추가해야 합니다.

.단계
. 왼쪽 탐색 창에서 *리소스*를 선택한 다음 목록에서 PostgreSQL 클러스터용 SnapCenter 플러그인을 선택합니다.
. 리소스 페이지의 * 보기 * 목록에서 리소스 유형을 선택합니다.
. (선택 사항) image:../media/filter_icon.gif["필터 아이콘"]를 선택하고 호스트 이름을 선택합니다.
+
그런 다음 image:../media/filter_icon.gif["필터 아이콘"]를 선택하여 필터 창을 닫을 수 있습니다.

. 클러스터를 선택한 다음 *자격 증명 구성*을 선택합니다.
. SnapCenter에서 PostgreSQL 클러스터 사용자용으로 구성된 자격 증명을 선택하십시오. 자격 증명에는 데이터 보호 작업을 수행하는 데 필요한 충분한 권한이 있는 유효한 사용자 이름과 암호가 포함되어야 합니다.
. OK * 를 선택합니다.


토폴로지 페이지에서 클러스터 구성을 수정할 수 있습니다.
